소개

Park's Emotion을 만든 저와 함께하는 여정에 대해 사람들에게 알릴 수 있는 좋은 기회입니다.

저는 박찬호이고, 현재 국립공주대학교 4학년 입니다. [간략한 배경 소개]

[포기하지 않는 청년]

전적 대학(충남도립대학) 2학년 때, 교내 대회를 하였습니다. 처음엔 안드로이드 앱을 하려고 하였으나 부족한 부분이 너무 많기에 CSS와 JavaScript를 이용하여 사이트를 만들기 시작하였습니다.
많이 부족한 실력임에도 불구하고 여러 가지 방법과 코드를 적용하는 방식을 서칭하고 실험해보면서 웹 사이트를 구축하기 시작했습니다.
그렇게 하여 여러 학과들이 참여한 곳에서 제가 처음으로 만든 웹 사이트를 이용하여 대회에 출전하였고, 부족한 실력임에도 불구하고 10등이라는 성적을 거두었습니다.이것을 기반으로 FrontEnd라는 꿈을 확고하게 되었습니다.

[책임감 있는 청년]

어렸을 때 부터 아버지는 저에게 책임감을 강조하셨습니다. 대신 제가 어떠한 일을 하고 그것을 진심으로 하고싶다면 그것에 대한 책임감을 강조하셨습니다.
이런 아버지의 말씀을 마음에 새겨넣고 저에게 주어진 일에 대해 최선을 다했고, 그 결과 여러 가지 직책을 맡았습니다.
조별과제가 주어졌을 땐 조장을 학과에서는 부학회장을 팀 프로젝트를 할 때는 팀장을 맡았습니다.
그 결과 홀로 가졌던 것보다 더욱 더 큰 책임감을 가져야 했으며 때론 지치고 포기하고싶었습니다.하지만 이를 통해서 더 많은 배움을 얻을 수 있었고, 책임감을 가지고도 부담스러워하지 않고 웃으면서 열심히 할 수 있는 청년이 되었습니다.

[Park's Emotion은 웹 사이트에 생체 데이터를 이용하여 보안을 실시하고 사람의 얼굴을 사용하여 감정 확인 및 패션 추천시스템입니다]

Python에 CNN과 JavaScript 등을 활용하여 사람의 감정을 확인하는 성능을 더욱 향상시켰습니다.

수행 업무

  • ResNet 2개를 이용한 안면마비 유무 판별 및 감정 확인 시스템
    • ResNet 두 개를 이용하여 안면마비의 유무 판별
    • 안면마비가 있으면 그 Data를 기반으로 감정 유추
    • 마비가 없는 사람일 경우 그 자체 Data를 기반으로 감정 유추
    • 모델 구성 및 확인
      1. 안면마비 유무 판별
        • 모델 선택 : ResNet101을 사용하였습니다.
        • 표준화 및 정규화 : 각 픽셀의 평균을 0,표준편차 1로 조정하고 정규화를 수행하였습니다.
        • 옵티마이저 : Adam이나 SGD를 사용할 예정입니다.
        • 정규화 방법 : Dropout이나 Batch Normalization을 사용할 예정입니다.
      2. 감정 판별
        • 안면마비 있음/없음에 따른 데이터 분리
        • 모델 설계
        • 데이터 증강
        • 학습 전략 : Transfer Learning을 해볼 생각입니다.
      3. 평가 및 조정
        • 평가 지표 : 정확도,F1 점수, 정밀도 등 여러가지 평가 지표를 사용할 예정입니다.
        • 조정 : 실험을 반복하면서 모델 구조,HyperParameter,데이터 전처리 방법 등을 조정할 예정입니다.
  • CLIP을 이용한 의류 추천 시스템
    • 모델 구성 및 확인
      1. 모델 구성
        • CLIP모델 활용 : OpenAI가 개발한 CLIP을 사용할 예정입니다.
        • Text형식의 질문과 Image를 입력받아 두 입력 사이의 상호작용을 학습시킬예정입니다.
        • 이 과정에서 얼굴 사진은 사용자의 스타일,성별,연령 등을 반영할 수 있도록 추가 정보를 추출하는 데 사용할 예정입니다
      2. 추천 시스템 구축
        • 특징 추출
        • CLIP모델을 사용하여 의류 이미지와 사용자 질문의 특징을 추출합니다.
        • 의류 데이터셋에 대해 미리 특징을 추출해놓고 사용자 질문이 들어올 때마다 해당 질문의 특징을 실시간으로 추출하여 매칭할 예정입니다.
        • 유사도 측정
        • 코사인 유사도, 유클리드 거리 등 사용할 예정입니다.
        • 추천 알고리즘
        • 계산된 유사도를 바탕으로 사용자에게 가장 적합한 의류를 추천합니다.
        • 여기서 유사도가 높은 상위 N개를 추천하는 방법으로 진행해볼예정입니다.
      3. 평가 및 조정
        • 성능 평가
        • 사용자로부터 피드백을 받아 추천 시스템의 성능을 평가하는 것으로 해볼 예정입니다.
        • 모델 조정
        • 추천 알고리즘의 파라미터 조정, 데이터 전처리 방법 개선,추가적인 데이터 수집 등
  • Regist Data Processing
    • Database와 연동하여 실행하는 것을 기반으로 수정 예정
